## 📌 Project Description & Purpose

This project is a **full-stack product feedback application** where users can submit, view, and filter feedback based on categories like UI, UX, Feature, Bug, and Enhancement.

The goal of this app is to simulate a real-world feedback system where users can share ideas and improvements, and developers can organize and manage those suggestions efficiently.

## ✨ Features

- 📥 Add new feedback (title, category, description)  
- 📋 View all feedback from the database  
- 🔍 Filter feedback by category (UI, UX, Feature, Bug, Enhancement)  
- ⚡ Dynamic updates when feedback is added  
- 🧠 Full backend + database integration  

---

## 🛠️ Tech Stack

### **Frontend**
- JavaScript, HTML, CSS  
- React (Vite)  
- React Router  
- Netlify  

### **Backend**
- Node.js  
- Express  
- PostgreSQL (Neon)  
- Render  

---

## 📡 API Endpoints

- `GET /get-all-suggestions`  
- `GET /get-suggestions-by-category/:category`  
- `POST /add-one-suggestion`  

---

## 🚀 What I Did

- Built a full-stack application using **React, Express, and PostgreSQL**
- Created REST API endpoints to fetch and store feedback data
- Connected the backend to a cloud database using **Neon PostgreSQL**
- Implemented dynamic filtering by category on the frontend
- Managed state using React hooks (`useState`, `useEffect`)
- Set up a Vite proxy for local development API calls
- Deployed the backend to **Render** and connected it to the frontend
- Debugged real-world issues like API routing, environment variables, and deployment errors
